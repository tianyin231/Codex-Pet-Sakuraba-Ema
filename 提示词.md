# Sakuraba Ema Codex Pet 生成提示词

目标：把原角色立绘转换成可爱的 Q 版 Codex pet。每个阶段下面的提示词都可以单独复制使用，不需要再手动拼接全局规则。

核心原则：Q 版只改变比例和可爱度，不改变角色身份。所有状态都必须保持同一个角色、同一套 Q 版头身比、同一画风、同一配色和同一套服装配饰。

## 0. 输出规格和帧数

- 最终 atlas：8 列 x 9 行。
- 单元格：192x208 px。
- 最终尺寸：1536x1872 px。
- 每次只生成一条横向 sprite row，不要一次生成整张 atlas。
- 所有生成图使用纯色 chroma-key 背景，不要透明棋盘格、白底、黑底、场景或地面。
- 状态顺序固定为：idle, running-right, running-left, waving, jumping, failed, waiting, running, review。

| Row | State | 帧数 |
| --- | --- | ---: |
| 0 | idle | 6 |
| 1 | running-right | 8 |
| 2 | running-left | 8 |
| 3 | waving | 4 |
| 4 | jumping | 5 |
| 5 | failed | 8 |
| 6 | waiting | 6 |
| 7 | running | 6 |
| 8 | review | 6 |

## 1. 通用上传规则

### 生成 Base 基准图

上传：

1. 原立绘

### 生成状态行

上传：

1. 原立绘
2. 已生成并确认过的 Base 基准图
3. 对应状态的 layout guide

layout guide 只作为布局参考。生成结果中不能出现 guide 的边框、色块、中心线、文字、编号、背景或任何 guide marks。

## 2. 最简单执行顺序

1. 原立绘 -> Base
2. 原立绘 + Base + idle layout guide -> idle
3. 原立绘 + Base + running-right layout guide -> running-right
4. running-right 可逐帧镜像 -> running-left
5. 如果不能镜像，原立绘 + Base + running-left layout guide -> running-left
6. 原立绘 + Base + waving layout guide -> waving
7. 原立绘 + Base + jumping layout guide -> jumping
8. 原立绘 + Base + failed layout guide -> failed
9. 原立绘 + Base + waiting layout guide -> waiting
10. 原立绘 + Base + running layout guide -> running
11. 原立绘 + Base + review layout guide -> review

running-left 只有在镜像不会破坏角色身份时才镜像。镜像前检查单边发饰、刘海方向、衣服图案、徽章、武器、配饰、光照和其他不对称特征。镜像时必须逐帧镜像并保持帧顺序，不要把整条 strip 反向排列。

## 13. 生成后检查清单

- 9 个状态都存在，顺序是 idle, running-right, running-left, waving, jumping, failed, waiting, running, review。
- 帧数分别是 6, 8, 8, 4, 5, 8, 6, 6, 6。

- running 是工作中，不是跑步。
- idle 有微变化，但不能像 waving、running、review 或 waiting。
- waving 只靠肢体动作表达，不添加挥手线。
- jumping 只靠身体高度和姿态变化表达，不添加地面或阴影。
